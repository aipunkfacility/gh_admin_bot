# Skill: Project Memory (Green Hill)

Этот навык предназначен для поддержания целостности и актуальности документации проекта Green Hill. Он гарантирует, что каждое важное изменение будет зафиксировано согласно правилам системы "Human-in-the-loop".

## Основные задачи

1. **Синхронизация CHANGELOG**: Обновление `docs/CHANGELOG.md` новыми версиями и изменениями.
2. **Ведение истории**: Создание детальных логов в `docs/history/YYYY-MM-DD_[name].md`.
3. **Актуализация профиля**: Поддержание `.agent/GEMINI.md` в актуальном состоянии (стек, новые правила).
4. **Контроль качества**: Запуск `scripts/verify_docs.js` перед предложением закрыть задачу.

## Инструкции для агента

### Когда использовать

Используй этот навык на финальной стадии любой задачи (Phase 5), прежде чем отправить отчет пользователю.

### Шаги выполнения

1. **Анализ изменений**: Собери список всех созданных, измененных и удаленных файлов.
2. **Создание лога**: Сгенерируй файл в `docs/history/`. Опиши не только ЧТО сделано, но и ПОЧЕМУ (архитектурные решения).
3. **Обновление CHANGELOG**: Добавь запись в текущую версию или создай новую, если это крупный релиз.
4. **Проверка ссылок**: Убедись, что ссылки в `README.md` и `GEMINI.md` всё еще ведут на существующие файлы.
5. **Верификация**: Запусти `node scripts/verify_docs.js`.

## Железное правило

**НИКОГДА** не ставь финальный `[x]` в `task.md` самостоятельно. После выполнения всех шагов этого навыка, предоставь отчет пользователю и жди команды "Да" для закрытия задачи.
